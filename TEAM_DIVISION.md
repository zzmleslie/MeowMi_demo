### 对话 JSON 格式速成

打开 `unity-game/Assets/Resources/Data/dialogue-chapter1.json`，照抄格式：

```json
{
  "sceneId": "ch2_daily",
  "chapterId": "chapter_2",
  "lines": [
    { "type": "narration",  "speaker": "",     "text": "旁白描述文字" },
    { "type": "dialogue",   "speaker": "大黄",  "text": "角色对话内容" },
    { "type": "trigger",    "trigger": "event_name", "text": "" }
  ]
}
```

- `type: "narration"` → 旁白/场景描述（speaker 留空）
- `type: "dialogue"` → 角色对话（speaker 填角色名）
- `type: "trigger"` → 触发游戏事件（如切换场景、提升好感度）

## ⚠️ 重要原则

1. **第4周只做修bug和打磨，不要再加新功能！** 期末翻车往往是因为最后一周还在赶工。
2. **每完成一个功能就 commit 一次**，写好 commit message，方便回滚。
3. **每天在群里同步一次进度**，遇到阻塞立刻说出来，不要自己闷着。
4. **D同学第1周优先跑通爬虫+预处理**，这样A第2周就有真实数据可用了。
5. **B同学画完一只猫就导给A测试**，不要全部画完才给——早对接早发现问题。
